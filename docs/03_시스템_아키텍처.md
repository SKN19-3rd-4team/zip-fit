# 시스템 아키텍처

(추후 Miro를 활용하여 별도 작성 예정)

## 1. 전체 아키텍처 개요

### 시스템 구성
- Frontend (Vue 3)
- Backend (FastAPI)
- Database (PostgreSQL + pgvector)
- AI Models (Embedding, Reranker, LLM)

---

## 2. 주요 컴포넌트

### Frontend
- Vue 3 애플리케이션
- AI 채팅 인터페이스
- 공고 리스트 조회
- 대시보드 통계

### Backend
- FastAPI RESTful API
- RAG 파이프라인
- 벡터 검색 시스템
- 세션 관리

### Database
- PostgreSQL (RDBMS)
- pgvector (벡터 확장)
- 3개 주요 테이블 (announcements, announcement_files, document_chunks)

### AI Models
- BAAI/bge-m3 (임베딩 모델)
- Dongjin-kr/ko-reranker (Reranker 모델)
- GPT-4o-mini (LLM)

---

## 3. 데이터 흐름

### 질의 응답 프로세스
1. 사용자 질문 입력
2. 질문 재구성 및 멀티쿼리 생성
3. 벡터 검색 및 키워드 검색
4. 검색 결과 재순위화
5. 컨텍스트 구성
6. LLM 답변 생성
