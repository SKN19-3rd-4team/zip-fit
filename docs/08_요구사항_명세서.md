## 2. 프로젝트 개요

### 2.1. 프로젝트 목적

ZIPFIT은 LLM과 RAG 기술을 활용하여 공공주택 공고를 통합하고, 사용자 맞춤형 주거 정보를 제공하는 AI 에이전트 서비스입니다.

### 2.2. 핵심 가치 제안

| 항목 | 설명 |
|------|------|
| **문제 해결** | 복잡한 공공주택 공고문을 쉽게 이해할 수 있도록 AI가 설명 |
| **맞춤형 추천** | 사용자 조건에 맞는 공고를 자동으로 추천 |
| **원스톱 서비스** | 공고 검색부터 신청, 계약, 대출까지 하나의 플랫폼에서 완결 |
| **정보 접근성** | 모든 국민이 공평하게 공공주택 정보에 접근 가능 |

### 2.3. 프로젝트 범위

#### 2.3.1. 프로토타입 범위 (현재 구현)

| 구분 | 범위 | 비고 |
|------|------|------|
| **데이터** | LH 공고 중 서울/경기 지역만 | GH, SH 크롤링 완료했으나 미사용 |
| **공고 유형** | 임대 및 분양 공고 | 14개 임대 유형, 2개 분양 유형 |
| **핵심 기능** | AI 상담 → 공고 비교/추천 → 신청 안내 | 기본 RAG 파이프라인 구현 |
| **제외 기능** | 자격 진단, 계약 지원, 대출 정보 제공 | 향후 확장 예정 |

#### 2.3.2. 최종 목표 서비스 흐름

```
AI 상담 → 자격 진단 → 공고 비교/추천 → 신청 안내 → 계약 지원 → 대출 정보 제공
```

### 3.2. 상세 기능 요구사항

#### FR-001: AI 상담 (대화형 인터페이스)

**설명**: 자연스러운 대화를 통해 사용자의 주거 니즈를 파악하고 공고 정보를 제공합니다.

**프로토타입 구현 상태**: ✅ 완료

**주요 기능**:
- 사용자 질문 입력 및 답변 생성
- 대화 히스토리 관리
- 맥락 인식 대화 (이전 대화 참조)
- 마크다운 형식 답변 지원

**입력**:
- 사용자 질문 (자연어)
- 사용자 ID (세션 식별)

**출력**:
- AI 답변 (텍스트, 마크다운 지원)
- 참조 공고 정보 (출처)
- 메타데이터 (질문 분석 정보)

**구현 위치**:
- Frontend: `front-end/zf_front/src/views/AiView.vue`
- Backend: `back-end/zip_fit/router.py`, `chatting.py`

**API 엔드포인트**:
- `POST /api/v1/chat`

#### FR-002: 질문 재구성 및 분석

**설명**: 사용자의 모호한 질문을 검색에 적합한 형태로 변환하고, 지역, 유형, 카테고리 등 필터 정보를 추출합니다.

**프로토타입 구현 상태**: ✅ 완료

**주요 기능**:
- 질문 재구성 (Query Rewriting)
- 멀티쿼리 생성 (Multi-Query Generation)
- 맥락 분석 (Context Analysis)

**입력**:
- 사용자 질문
- 대화 히스토리 (선택)

**출력**:
- 재구성된 질문
- 추출된 필터 정보 (지역, 유형, 카테고리)
- 검색 키워드
- 멀티쿼리 목록

**구현 위치**:
- `back-end/zip_fit/llm_handler.py`

**LLM 모델**: GPT-4o-mini

#### FR-003: RAG 기반 문서 검색

**설명**: 공고 PDF 문서를 벡터화하여 저장하고, 사용자 질문과 관련된 문서를 검색합니다.

**프로토타입 구현 상태**: ✅ 완료

**주요 기능**:
- 하이브리드 검색 (벡터 검색 + 키워드 검색)
- 재순위화 (Reranking)
- 청크 병합 (Chunk Merging)
- 컨텍스트 구성 (Context Building)

**검색 전략**:
- 벡터 검색: 의미 기반 유사도 검색 (BAAI/bge-m3)
- 키워드 검색: 정확한 단어 매칭
- 재순위화: Cross-Encoder 기반 (Dongjin-kr/ko-reranker)

**구현 위치**:
- `back-end/zip_fit/gongo.py`

**데이터베이스**: PostgreSQL + pgvector

## 4. 비기능 요구사항

### 4.1. 성능 요구사항

| 항목 | 요구사항 | 현재 상태 | 비고 |
|------|----------|----------|------|
| **응답 시간** | 평균 3초 이내 | 측정 필요 | LLM 호출 시간 포함 |
| **동시 접속자** | 최소 10명 동시 처리 | 지원 | 비동기 처리 활용 |

| **데이터베이스 쿼리** | 평균 200ms 이내 | 측정 필요 | 비동기 쿼리 최적화 |

### 4.2. 확장성 요구사항

| 항목 | 요구사항 | 현재 상태 |
|------|----------|----------|
